Assignment: Software Engineer (Full Stack) 
Task: Build a Simple Task Manager 
Objective 
This assignment is designed to assess your ability to build a basic Full Stack web application 
with CRUD functionality. 
The Goal 
Create a web application where users can create, view, update, and delete their tasks. 
1. Functional Requirements 
A. Frontend (User Interface) 
• Dashboard: Display list of all tasks with Title, Description, Status, Created Date. 
• Add Task Form with validation (title required). 
• Edit task functionality. 
• Delete tasks. 
• Mark task as Completed / Pending. 
B. Backend (API) 
Build a RESTful API using Node.js + Express. 
• GET /tasks - Get all tasks 
• POST /tasks - Create a new task 
• PUT /tasks/:id - Update task 
• DELETE /tasks/:id - Delete task 
C. Database 
Use MongoDB with Mongoose. 
Schema fields: title, description, status (Pending/Completed), createdAt. 
D. Business Logic 
• Tasks marked Completed should display a visual indicator. 
• Tasks should be sorted by latest created first. 
2. Technical Requirements 
Frontend: React.js, Hooks, Tailwind or Bootstrap, Responsive design. 
Backend: Node.js, Express.js, MongoDB, Basic MVC folder structure. 
3. Bonus Features 
Optional improvements: Task search, Pagination, Simple authentication. 
4. Timeline & Submission 
Duration: 2-3 Days. 
Submit via GitHub repository including source code, README with setup instructions, and 
optional demo video. 
Evaluation Criteria 
Full Stack connectivity, Code quality, Error handling, UI/UX.
